# Security policy

## Supported version

Security fixes are applied to the latest public release and the `main` branch.

## Reporting a vulnerability

Do not open a public issue for an unpatched vulnerability, exposed credential,
unsafe filesystem operation, or release-pipeline weakness. Use GitHub's
**Report a vulnerability** private-reporting feature on this repository.

Include the affected version, reproduction steps, impact, and any suggested
mitigation. The maintainer will acknowledge a complete report as soon as
practical and will coordinate disclosure after a fix or mitigation exists.

Never send real Civ V logs or project archives containing personal information
unless specifically requested through a private channel. Remove usernames,
local paths, tokens, and unrelated mod content from evidence.
